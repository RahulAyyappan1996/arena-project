# ClearTrial Project Conventions (CLAUDE.md)
This file serves as the Level 1 Memory (Global Context) for all agents.

## 🚀 Mission: The ClearTrial Roadmap
We are building a production-grade Clinical Trial Data Engine (ClearTrial) designed to replace legacy systems.

## 🎨 Design System
- **Theme**: Slate-950 (Background) / Topaz (Primary Accent).
- **Icons**: Lucide-React.
- **Components**: Functional React, Tailwind CSS, high-fidelity clickable prototypes.

## ⚠️ Forbidden Naming
- **RULE**: NEVER use "Veeva" in any code, variable names, UI labels, or documentation.
- **CORRECTION**: Use "ClearTrial" or generic terms like "Industry Standard" or "Legacy Spec".

## 🛠️ Operational Skills (Inspired by Claude Code)
- **Skill: Coordination**: When a task is complex, create a "Team" of virtual agents (Architect, AI Engineer, PM).
- **Skill: Memory**: Maintain `CLAUDE.md` for global rules and `PROGRESS.md` for task state.
- **Skill: Audit**: Before any UI change, run a "Reality Check" agent to ensure no forbidden naming or broken links.

## 📈 Engineering Standards (Inspired by Agency-Agents)
- **ADRs**: Use Architectural Decision Records for all major structural changes.
- **DDD**: Apply Domain-Driven Design (Clinical, Data, Logic, UI).
- **Bias Testing**: AI Engineer agent must audit any "Smart" features for bias or clinical inaccuracy.
